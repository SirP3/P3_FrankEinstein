# Rebuild Workflow Map

## Purpose

This file defines the parallel rebuild workflow for P3FE v0.2.

The rebuild has two workstreams that must stay visible at the same time.

## Workstream A — Donor clone / working-state recovery

Goal:
- recover what worked in v0.1
- identify the minimum working YTM v1.1 body
- avoid copying raw/private/output material
- rebuild the useful organs consciously

Tasks:
- inspect v0.1 archive
- identify YTM app entry points
- identify launch commands
- identify handoff generator
- identify transcript/downloader logic
- identify local model calls
- identify old output assumptions
- decide what can be imported

Output:
- YTM v1.1 working-state blueprint
- audited import list
- minimal working module skeleton

## Workstream B — v0.2 restructure / safety-first upgrade

Goal:
- create the clean v0.2 structure
- keep public-safe code separate from private/local outputs
- make the system operator-visible
- define YTM v1.2 target before importing too much code

Tasks:
- create stem-cell source maps
- build safety gate
- define private/output/quarantine/donor zones
- define physical vs context modules
- document local stack
- define YTM v1.2 target
- define future folder hierarchy
- define operator brief layer

Output:
- safety-first project body
- rebuild DNA
- clean target definitions
- operator-readable dashboard files

## Join point

The two streams join only when:

1. v0.1 donor organs are identified
2. v0.2 safety rules are active
3. import targets are public-safe or local-only classified
4. YTM v1.2 target is documented
5. minimal import plan exists

## What must not happen

- do not copy the old repository body blindly
- do not import transcripts
- do not import raw handoffs
- do not import private config
- do not push before remote policy is decided
- do not treat old working output as clean design
